You are helping the user add a new photo album to this portfolio project. Follow these steps carefully and interactively.

## Step 1: Gather album details

Ask the user for:
1. **Album name** — the display title for the album (e.g., "Bandits FC", "Icarus Cup")
2. **Number of palette colors** (`image_color_count`) — how many images to sample for the gradient palette generator (default: 5)

Wait for the user's response before proceeding.

## Step 2: Find the image folder

Look for a folder matching the album name inside `src/assets/photos/`.

- List all folders in `src/assets/photos/` and try to match the album name using case-insensitive, hyphenated, and spaced variations (e.g., "Icarus Cup" → `icarus-cup`, `Icarus-Cup`, `icarus cup`, etc.)
- **If a match is found**, confirm it with the user and use that exact folder name as the album `id`.
- **If no match is found**, tell the user which folders exist and ask them to type the correct folder name. Then continue.

## Step 3: Scan images and get dimensions

- List all image files (`.jpg`, `.jpeg`, `.png`) in the matched folder.
- For each image, get its pixel dimensions using `sips -g pixelWidth -g pixelHeight "<path>"`.
- Determine a camelCase import prefix from the folder name (e.g., `icarus-cup` → `imgIcarus`, `bandits` → `imgBandits`). Follow the naming convention in `src/data/photos.ts`.

## Step 4: Write imports and album entry in `src/data/photos.ts`

Read the existing `src/data/photos.ts` file. Then:

1. **Add JPEG imports** at the end of the "Original JPEG imports" section (before the "Responsive WebP srcSet imports" comment). Follow the exact pattern:
   ```ts
   import imgPrefix1234 from '../assets/photos/<folder>/filename.jpg';
   ```

2. **Add srcSet imports** at the end of the "Responsive WebP srcSet imports" section (before the `export interface` line). Use the image's largest dimension as the final width breakpoint. Follow the exact pattern:
   ```ts
   import imgPrefix1234SrcSet from '../assets/photos/<folder>/filename.jpg?w=480;768;1024;<largestDim>&format=webp&as=srcset';
   ```
   For the last `w` value, use `width` if landscape or `height` if portrait (the larger dimension of the image).

3. **Add the album object** to the `albums` array (before the closing `];`). Follow this structure:
   ```ts
   {
     id: '<exact-folder-name>',
     title: '<Album Name from user>',
     description: '<Ask user or generate a short description>',
     cover: { src: <firstImport>, srcSet: <firstImportSrcSet>, alt: '<Album Name> cover photo', width: <w>, height: <h> },
     photos: [
       { src: <import>, srcSet: <importSrcSet>, alt: '<Album Name> photo description', width: <w>, height: <h> },
       // ... one entry per image
     ],
     tags: ['<ask user or infer>'],
   },
   ```
   Note: Do NOT add a `gradient` field — that will be generated by the palette script in the next step.

4. **Ask the user** for:
   - A short album `description` (or suggest one based on the album name)
   - `alt` text approach — whether to use generic alts like "<Album Name> photo 1" or if they want to provide specific alts for each image
   - `tags` — a few lowercase tags for the album

## Step 5: Update sitemap.xml

Add a new `<url>` entry to `public/sitemap.xml` for the new album:
```xml
<url>
  <loc>https://ashujainvi.com/photos/<album-id></loc>
  <changefreq>monthly</changefreq>
  <priority>0.7</priority>
</url>
```

## Step 6: Run the palette generator

Run the palette generator script to analyze the album's images and add the `gradient` field:
```bash
npx tsx scripts/albumPaletteGenerator.ts <folder-name> <image_color_count>
```

Report the generated gradient colors to the user.

## Step 7: Verify

- Confirm the album was added correctly by reading back the album entry from `src/data/photos.ts`.
- Remind the user to check the album in the browser at `/photos/<album-id>`.
